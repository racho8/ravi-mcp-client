Request Flow Architecture

User Input

User types natural language command in the chat interface. Example: "Show all products grouped by category"

Route Handler

Express.js processes the request, validates input, and orchestrates the flow between components

LLM Client

Ollama LLM interprets user intent and converts natural language to structured tool invocations

Schema Cache

Cached MCP tool schemas provide available operations and parameters for efficient processing

MCP Client

Handles authentication and communicates with remote MCP server using JSON-RPC protocol

Remote MCP Server

Google Cloud Run service processes requests, manages product database, and returns results

Response Processing

Route handler applies business logic like grouping, filtering, counting, and duplicate detection

UI Formatting

Frontend formats and displays results with beautiful cards, grouping, and interactive elements
